# 수다쟁이 모자 Minecraft Mod 개발 계획서

## 1. 프로젝트 개요

### 컨셉
플레이어가 장비할 수 있는 "수다쟁이 모자" 아이템을 추가하는 Fabric 모드.
이 모자는 단순 장비가 아니라, 플레이어와 대화하고 주변 상황에 반응하며, 기억을 축적하는 동료형 캐릭터다.

### 핵심 방향
- 최신 Minecraft + Fabric 지원
- 우선 타겟: **싱글플레이**
- 우선 LLM 백엔드: **Ollama**
- 인게임 설정 UI 제공
- 모자의 페르소나 설정 가능
- 플레이어 발화에 대한 반응 + 주변 상황에 대한 능동 발화
- 플레이어와의 관계성을 높이는 메모리 시스템 제공
- **대화 언어 설정 가능**

### 한 줄 목표
"플레이어를 기억하고, 세계를 함께 체험하며, 상황에 맞게 떠드는 동료 모자 만들기"

---

## 2. 기술적 타당성 검토 결론

### 결론
**기술적으로 구현 가능하다.**

특히 아래 조건으로 범위를 제한하면 현실성이 높다.
- 멀티플레이 제외, 싱글플레이 우선
- 외부 상용 API 대신 로컬 Ollama 우선
- 화려한 UI보다 텍스트 기반 상호작용 우선
- 능동 대화는 전부 LLM 호출이 아니라 로컬 규칙 + 선택적 LLM 호출 조합

### 주요 난점
1. 모자가 너무 자주 말하지 않도록 제어하는 UX 설계
2. 메모리를 "기억하는 느낌"이 들게 하면서도 가볍게 유지하는 구조
3. Ollama 응답 지연과 비동기 처리
4. 페르소나 유지와 상황 적합성의 균형
5. 다국어/언어 설정을 포함할 때 프롬프트와 UI 일관성 유지

---

## 3. 제품 비전

### 플레이 경험
플레이어는 모자를 착용한 뒤, 단순히 질문-응답 챗봇을 넘어서 다음 경험을 하게 된다.
- 초반 생존 시 모자가 조언해줌
- 위험 상황에서 짧고 즉각적으로 경고함
- 플레이어가 말을 걸면 캐릭터성 있게 대답함
- 지난 대화와 사건을 기억해 관계성이 생김
- 플레이어가 설정한 언어와 성격으로 일관되게 말함

### 차별점
- 일반 채팅봇이 아니라 **착용형 동료 아이템**
- 게임 이벤트와 실제 플레이 맥락을 반영
- 월드 내 경험을 기억하는 캐릭터
- 로컬 LLM 기반으로 프라이버시와 실험성이 좋음

## 5. 핵심 기능 정의

## 5.1 모자 아이템
- 플레이어가 장비 가능한 모자 아이템
- 기본적으로 헬멧 슬롯 장착 방식 우선
- 장착 시에만 기능 활성화
- 해제 시 대화/능동 반응 비활성화 또는 일시 정지

## 5.2 대화 기능
- 플레이어가 말을 걸면 모자가 응답
- 응답은 채팅창 중심으로 출력
- 필요 시 액션바/토스트로 짧은 경고 표시
- 모자 이름, 말투, 캐릭터성 유지

### 입력 방식 후보
1. 일반 채팅 감지
2. 전용 명령(`/hat say ...`)
3. 전용 UI 대화창

### 추천 초기안
- 디버깅과 구현 안정성을 위해 **전용 명령 또는 전용 키/UI 기반 입력** 우선
- 일반 채팅 후킹은 후속 단계에서 선택적으로 지원

## 5.3 능동 발화
모자는 플레이어가 직접 말을 걸지 않아도 특정 상황에서 먼저 말할 수 있다.

### 능동 발화 트리거 예시
- 첫 스폰
- 첫 밤 도래
- 체력 저하
- 배고픔 심화
- 몬스터 접근
- 새로운 바이옴 진입
- 의미 있는 자원 발견
- 사망 직전/직후
- 오랜 침묵 이후 분위기 환기

### 구현 원칙
- 긴박한 경고는 규칙 기반 우선
- 캐릭터성 있는 긴 대사는 LLM 호출
- 쿨다운과 반복 억제 필수

## 5.4 페르소나 설정
- 프리셋 제공
  - 초보자 가이드형
  - 수다 많은 친구형
  - 겁많은 생존 코치형
  - 허풍쟁이 마법사형
- 커스텀 설명문으로 직접 페르소나 입력 가능
- 말투, 길이, 친밀도, 농담 비율 등 옵션화 가능

## 5.5 메모리 기능
- 최근 대화 기록 저장
- 중요한 사실 추출 및 장기 기억 저장
- 일정 시점마다 요약 메모리 생성
- 월드별 기억 유지

## 6. 언어 설정 설계

이 항목은 초기 요구사항에 **반드시 포함**한다.

### 목표
플레이어가 모자와의 대화 언어를 설정할 수 있어야 한다.

### 지원 방향
- UI에서 대화 언어 선택 가능
- 최소 1차 지원:
  - 한국어
  - 영어
  - 자동(auto)
- 후속 확장:
  - 일본어
  - 중국어
  - 기타 커스텀 locale

### 언어 설정이 영향을 주는 영역
1. 모자의 응답 언어
2. 위험 경고/가이드 문구
3. UI 설명 문구 일부(선택 사항)
4. 메모리 요약 언어
5. 페르소나 프롬프트 내 말투 규칙

### 추천 설계

#### 옵션 구조 예시
- `chatLanguage`: `ko` | `en` | `auto`
- `systemLanguage`: UI 표시 언어, 초기에는 게임 언어 따르기
- `allowMixedLanguage`: 혼합 언어 허용 여부

#### 동작 방식
- `ko`: 항상 한국어 중심으로 응답
- `en`: 항상 영어 중심으로 응답
- `auto`: 플레이어 최근 발화 언어 + 게임 locale + 기존 대화 문맥을 바탕으로 결정

### 구현상 주의점
- 모델이 언어 지시를 무시할 수 있으므로 프롬프트에 명확히 강제해야 함
- 경고 메시지는 LLM 대신 로컬 템플릿 기반으로도 언어별 준비 필요
- 메모리를 여러 언어로 저장하면 혼란스러울 수 있으므로 내부 저장 언어 전략 필요

### 메모리 언어 전략 추천
- 내부 메모리 fact는 구조화 데이터(JSON key-value)로 언어 중립적으로 저장
- 요약 텍스트는 선택 언어 기준으로 생성
- 긴급 경고 템플릿은 다국어 로컬 리소스로 분리

### 언어 관련 MVP 범위
- 한국어 / 영어 / 자동
- UI에서 변경 가능
- 응답 언어 강제 프롬프트 적용
- 로컬 경고 문구도 한국어/영어 제공

## 7. 권장 시스템 아키텍처

## 7.1 상위 구조
1. Hat System
2. Event Sense System
3. Dialogue Engine
4. Ollama Client
5. Memory System
6. UI / Config System

## 7.2 모듈별 역할

### 1) Hat System
- 모자 아이템 등록
- 장착 여부 추적
- 현재 활성 모자 상태 관리

### 2) Event Sense System
- 플레이어 상태 감지
- 월드 이벤트 감지
- 발화 후보 이벤트 생성
- 이벤트 중요도 및 중복 억제 처리

### 3) Dialogue Engine
- 최근 대화/메모리/상황 정보를 조합
- 페르소나와 언어 설정 적용
- LLM 요청 프롬프트 생성
- 응답 결과 후처리

### 4) Ollama Client
- localhost Ollama API 호출
- 모델 존재 여부 확인
- 타임아웃 및 에러 처리
- 비동기 요청 실행

### 5) Memory System
- 최근 대화 버퍼
- 중요한 사실 저장소
- 요약 메모리 저장
- 월드별 영속화

### 6) UI / Config System
- Ollama 연결 설정
- 모델 선택
- 언어 선택
- 페르소나 편집
- 수다 빈도 조절
- 메모리 보기/초기화

## 8. 상세 설계 방향

## 8.1 이벤트 처리 전략

### 핵심 원칙
모든 이벤트를 LLM에 보내지 않는다.

### 2단계 전략
1. **로컬 규칙 판단**
   - 지금 말해야 하는지 결정
   - 긴급도, 중요도, 최근 발화 여부 체크
2. **선택적 LLM 호출**
   - 캐릭터성 있는 응답이 필요한 경우만 호출

### 예시
- 크리퍼 접근 -> 즉시 로컬 경고
- 첫 밤 -> 짧은 로컬 안내 또는 짧은 LLM 반응
- 플레이어가 "무서워"라고 말함 -> LLM 응답
- 오랜 탐험 후 희귀 장소 발견 -> LLM 이벤트 대사

## 8.2 프롬프트 구성 요소
- 모자 기본 역할 설명
- 현재 페르소나 설명
- 대화 언어 지시
- 말 길이/톤 제한
- 현재 상황 요약
- 최근 대화 기록
- 장기 메모리 요약
- 금지 사항(너무 길게 말하지 말 것, 반복하지 말 것 등)

### 예시 규칙
- 항상 설정된 언어로 말하라
- 전투/위험 시에는 1~2문장 이내
- 플레이어를 도우려는 동료처럼 말하라
- 같은 내용 반복 금지
- UI/시스템 내부 용어를 드러내지 말라

## 8.3 응답 후처리
- 최대 길이 제한
- 공백/반복 문장 제거
- 언어 검사(선택적)
- 위험 상황에서는 긴 응답 잘라내기
- 과도한 메타 발언 제거

## 9. 메모리 설계

## 9.1 메모리 유형

### A. 단기 메모리
- 최근 플레이어 발화 N개
- 최근 모자 발화 N개
- 최근 이벤트 N개

### B. 장기 메모리
- 목표
- 선호
- 별명
- 자주 하는 행동
- 주요 성과
- 반복 실수

### C. 요약 메모리
- 최근 플레이 스타일 요약
- 관계성 변화
- 현재 월드의 장기 흐름

## 9.2 저장 단위
### 초기 추천
- **월드별 저장**

### 이유
- 월드마다 다른 서사가 생김
- 관리가 단순함
- 플레이어 입장에서 자연스러움

## 9.3 저장 포맷 예시

```json
{
  "worldId": "example-world",
  "chatLanguage": "ko",
  "personaId": "guide_friendly",
  "recentMessages": [],
  "facts": [],
  "summary": "",
  "cooldowns": {}
}
```

## 11. Fabric 구현 방향

## 11.1 초기 기술 방향
- 최신 Fabric 버전 타겟
- 기본 헬멧 슬롯 장비 방식 우선
- 데이터 저장은 world save 또는 config 기반 파일
- UI는 기본 Screen 기반 또는 설정 라이브러리 활용

## 11.2 주의할 점
- 네트워크/HTTP 호출은 메인 스레드에서 실행 금지

- 이벤트가 너무 자주 발생하지 않도록 debounce 필요

## 12. 개발 단계 계획

## Phase 0 — 사전 검증
### 목표
핵심 컨셉이 재밌는지 빠르게 확인

### 작업
- 최신 Fabric 개발 환경 구성
- 테스트 모자 아이템 추가
- Ollama 로컬 연결 테스트
- 간단한 프롬프트로 1회 응답 받기

### 산출물
- 장착 가능한 테스트 모자
- 개발 환경에서 Ollama 응답 성공

---

## Phase 1 — MVP 대화 구현
### 목표
"말하는 모자"의 최소 경험 완성

### 작업
- 모자 장비 여부 확인
- 플레이어 입력 수집 방식 구현
- Ollama 응답 출력
- 기본 페르소나 1~2개 적용
- 대화 언어 설정(ko/en/auto) 반영
- 최근 대화 버퍼 저장

### 완료 기준
- 모자를 쓰고 말 걸면 응답함
- 한국어/영어 설정 전환 가능
- 응답이 너무 길지 않고 캐릭터성이 있음

---

## Phase 2 — 능동 반응 추가
### 목표
게임 동료 같은 느낌 강화

### 작업
- 첫 밤 / 체력 저하 / 몹 접근 감지
- 규칙 기반 경고 템플릿 추가
